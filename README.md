# BiBiTool：B 站收藏夹智能整理工具

BiBiTool 是一个在本机运行的 B 站收藏夹整理工具。它可以一次选择多个已有收藏夹，汇总其中可访问的收藏资源并交给 AI 分类；执行前可以预览、手动调整和微调方案，确认后才会创建目标收藏夹并移动资源。

除智能整理外，项目还提供多账号管理、失效资源扫描与批量删除、空收藏夹批量删除、收藏夹排序等功能。

## 主要功能

### 智能整理

- 支持多选源收藏夹并批量整理，不再局限于单个收藏夹。
- 支持 B 站收藏夹返回的多种资源类型，不仅限于普通视频。
- 提供快速模式和完整模式：快速模式主要使用标题，完整模式还会获取简介和标签。
- 分类精细度支持粗略、均衡、精细和自定义，可限制最多生成的分类数量。
- AI 批次大小可在设置中调整，范围为 `10–200`。
- AI 返回无效、截断或漏项时会自动拆分批次并重试，尽量减少“未分类”条目。

### 方案预览与微调

- 按分类树形预览整理方案，分类和跳过条目均可折叠。
- 可按源收藏夹或分类筛选资源，并手动修改单条资源的分类。
- 可输入自然语言要求生成新方案，例如“把官方作品单独放在一个收藏夹内”。
- 微调过程显示阶段、百分比、已处理数量和重试次数，支持取消。
- 支持切换不同版本的预览方案，并可单独重试未分类条目。
- 最终执行时显示创建收藏夹和移动资源的实时进度，失败项可重试。

### 收藏夹管理

- 点击收藏夹可进入独立页面查看其中的资源列表。
- 支持单独删除空收藏夹，也可批量选择空收藏夹后统一删除。
- 支持拖动调整收藏夹顺序；进入排序模式后调整顺序，再明确保存到 B 站。
- 默认收藏夹受保护，不参与普通收藏夹删除。

### 账号与失效资源清理

- 支持扫码添加账号、切换账号和退出当前账号。
- 各账号的登录信息、整理会话和清理任务相互隔离。
- 可扫描当前账号全部收藏夹中的“已失效”和“无法访问”资源。
- 扫描结果按收藏夹分组，可筛选、折叠、全选或逐项选择后批量删除。
- 删除后会重新读取收藏夹确认结果，失败项会保留以便重试。

### 进度与恢复

- 拉取资源、AI 分类、方案微调、方案执行和失效扫描均提供进度反馈。
- 整理和扫描任务可在后台继续，返回页面后可恢复当前状态。
- 从设置或账号管理页返回时，会回到进入前的整理、预览或资源页面。
- 会话和方案保存在本地数据库中，关闭浏览器后仍可恢复未完成工作。

## 环境要求

- Windows 10/11
- Python 3.10 或更高版本
- 可以访问 B 站及所配置的 AI 接口
- 一个支持 OpenAI 兼容接口的模型服务，例如 DeepSeek、通义、Kimi 或其他兼容服务

## 安装与启动

首次使用先在项目目录安装依赖：

```powershell
python -m pip install -r requirements.txt
```

Windows 推荐双击 `启动.bat`。也可以运行 `start.bat`，两个脚本都会优先使用项目内的 `.venv`，否则使用系统 Python；若服务已经运行，则只打开浏览器，不会重复启动。

也可以手动启动：

```powershell
python main.py
```

服务默认只监听 `127.0.0.1:8765`，并自动打开 [http://127.0.0.1:8765](http://127.0.0.1:8765)。默认情况下局域网内其他设备无法连接，接口也会拒绝不受信任的 Host 和跨站写请求。

确实需要从局域网访问时，必须同时设置监听地址、至少 16 位的访问令牌和允许的 Host。例如在 PowerShell 中：

```powershell
$env:BIBITOOL_HOST = "0.0.0.0"
$env:BIBITOOL_LAN_TOKEN = "请替换为至少16位的随机令牌"
$env:BIBITOOL_ALLOWED_HOSTS = "192.168.1.20"
python main.py
```

程序会在本机打开一次性配对链接并把令牌写入 `HttpOnly` Cookie。其他设备首次访问也必须使用带 `access_token` 的配对链接；不要把配对链接或令牌发给不可信的人。`BIBITOOL_PORT` 可用于修改默认端口。

停止服务时，请关闭启动脚本对应的控制台窗口，或在控制台中按 `Ctrl+C`。仅关闭浏览器不会停止后台服务。

## 首次配置

1. 打开设置页，填写 AI 服务的 Base URL、API Key 和模型名称。
2. 根据模型能力设置每批分类条数，允许范围为 `10–200`。批次越大，调用次数通常越少，但响应过长或解析失败的风险也会增加。
3. 保存配置后扫码登录 B 站。
4. 登录成功后进入收藏夹选择页。

AI 配置保存在本机 `config.json` 中。API Key 留空保存时会保留原有密钥，不会用空字符串覆盖。没有完整 AI 配置时，已有方案查看、手动调整、执行、失效清理和账号管理仍可使用；只有首次分类、重试未分类和方案微调需要 AI。

## 使用流程

1. 在首页选择快速模式或完整模式。
2. 设置分类精细度；自定义模式可直接指定分类数量上限。
3. 勾选一个或多个源收藏夹，点击“开始智能整理”。
4. 等待程序拉取资源并完成 AI 分类，页面会持续显示实时进度。
5. 在预览页按收藏夹或分类查看方案，必要时手动调整、重试未分类或生成新方案。
6. 确认方案后执行。程序会创建目标收藏夹并移动资源，执行过程会显示当前进度。
7. 在完成页查看成功、失败和跳过数量；失败项可再次重试。
8. 整理后如产生空的源收藏夹，可在首页单独或批量删除。

目标收藏夹默认创建为私密收藏夹。源收藏夹中的同一资源只会按其实际来源位置处理，跨收藏夹重复收藏不会被误合并成一个删除位置。

执行任务会持久化目标收藏夹和每个“资源 + 来源收藏夹”的移动状态。程序在执行中断后会先核验源收藏夹与已创建的目标收藏夹，再继续尚未确认的位置，避免重复创建目标收藏夹或重放已经成功的移动。

## 失效资源清理

首页的“检查失效内容”会扫描当前账号的全部收藏夹，包括默认收藏夹：

- **已失效**：B 站返回的资源状态明确表明内容已经失效。
- **无法访问**：完整资源 ID 列表中存在该资源，但详情分页始终无法返回它。

扫描结果默认选中问题条目，但删除前仍会显示资源数量和涉及的收藏夹数量，并要求再次确认。删除属于真实的 B 站写操作，请核对后再执行。

## 本地数据与安全

- `config.json`：AI 接口配置，包含 API Key。
- `bilibili_cookie.json`：早期单账号登录信息，升级后会自动兼容处理。
- `accounts/<账号ID>/bilibili_cookie.json`：各 B 站账号的登录信息。
- `bibi.db`：账号、收藏夹缓存、整理会话、方案版本和清理任务。
- `bibi.db.backup-*`：数据库结构升级前自动生成的备份。

上述文件均可能包含敏感信息，已经加入 `.gitignore`。不要分享或提交这些文件，也不要把项目目录暴露到公网。

前端样式和图标均随项目本地提供，运行界面不依赖外部 CDN。数据库进行破坏性结构迁移前会生成带唯一时间戳的备份；确认升级正常后，可自行归档较旧的备份文件。

## 常见问题

### 页面打开后收藏夹需要等待几秒

程序正在从 B 站拉取收藏夹及资源数量，页面会显示加载动画。收藏夹较多或 B 站响应较慢时，等待时间会相应增加。

### AI 分类仍出现未分类

程序会自动处理非 JSON、响应截断和漏项，并对失败批次进行拆分重试。若多次重试后仍失败，可在预览页点击“重试未分类”，或适当减小设置中的 AI 批次大小。

### 生成新方案耗时较长

方案微调需要先分析现有分类与用户指令，再分批处理全部资源。进度面板会显示当前阶段和完成数量；筛选分类或折叠列表不会隐藏正在运行的进度。

### 删除收藏夹较慢

删除前后需要向 B 站校验收藏夹状态。批量删除空收藏夹可以减少重复操作，但仍会受到 B 站接口速度和频率限制影响。

### 浏览器关闭后仍有控制台窗口

这是正常的后台服务进程。请回到启动 BiBiTool 的控制台按 `Ctrl+C`，或直接关闭该控制台。不要仅根据“控制台窗口主机”名称结束进程，以免影响其他应用。

### 登录或接口失效

- B 站登录过期时，请在账号管理页重新扫码登录。
- AI 鉴权失败时，请检查 Base URL、API Key 和模型名称。
- 更新代码后应重启服务，浏览器刷新不会重新加载后端代码。

### 程序重启后任务显示为中断

- 采集阶段中断会从当前源收藏夹快照重新采集，不会把上次残留资源混入新结果。
- AI 分类中断会保留已采集资源并允许继续分类，不必重复拉取收藏夹。
- 最终执行中断会回到可继续执行状态；再次确认执行时会先进行远端对账。
- 清理或删除任务出现网络错误时会保存明确的失败状态，重试前会先核验已经从 B 站消失的条目。

## 手动验收建议

- 使用小收藏夹跑通多选、分类、预览、执行和失败重试流程。
- 在 AI 分类、方案微调和最终执行阶段确认进度会持续更新。
- 在预览页筛选分类、折叠分组，确认微调进度仍保持可见。
- 从整理过程进入设置和账号管理，再确认可以返回原页面。
- 进入收藏夹详情页核对资源数量和资源类型。
- 测试空收藏夹单独删除与批量删除，并确认默认收藏夹不可删除。
- 在排序模式中拖动收藏夹，保存后刷新并核对 B 站顺序。
- 使用测试收藏夹验证失效扫描、筛选、选择、删除确认和失败重试。
- 切换账号后确认不能读取另一个账号的会话或清理任务。
- 使用桌面浏览器和手机宽度检查按钮、进度和底部操作栏是否遮挡内容。

## 已知限制

- 项目使用的 B 站接口不是公开稳定的官方开放 API，参数和行为可能随时变化。相关调用集中在 `core/bilibili_api.py`。
- 创建、移动、排序和删除都会直接修改 B 站收藏夹，其中部分操作不可逆，建议先用小收藏夹验证。
- 完整模式、大批量 AI 分类和全账号失效扫描会产生较多网络请求，耗时受资源数量、AI 服务限流和 B 站响应速度影响。
- “无法访问”依据资源 ID 列表与详情分页的差异判断，删除前应人工核对标题、来源收藏夹和问题类型。
- 单次整理最多选择 100 个源收藏夹；批量删除请求一次最多选择 5000 个问题条目。大规模数据会自动分块查询，但耗时仍取决于 B 站和 AI 接口速度。
