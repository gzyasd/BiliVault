# 首页空收藏夹行内删除设计

## 目标

在首页收藏夹列表中，让用户直接删除空的普通收藏夹，同时保护默认收藏夹，并避免页面缓存过期造成误删。

## 交互设计

- 仅当 `media_count` 为 `0` 且 `fav_state` 不为 `1` 时，在收藏夹行右侧显示垃圾桶图标按钮。
- 默认收藏夹由 B 站返回的 `fav_state == 1` 判断，不根据标题推断；即使为空也不显示删除按钮。
- 点击垃圾桶按钮必须阻止事件冒泡，不得改变该收藏夹的多选状态。
- 删除前弹出确认框并显示收藏夹名称，明确提示操作不可逆。
- 删除期间禁用按钮并显示加载图标；成功后重新拉取收藏夹列表，失败则保留原行并显示错误原因。

## 后端安全边界

新增 `DELETE /api/folders/{fid}`。接口每次都通过 B 站重新拉取当前账号的收藏夹列表，并依次校验：

1. 收藏夹存在；
2. `fav_state != 1`，不是默认收藏夹；
3. `media_count == 0`，当前仍为空；
4. 当前没有正在运行的采集、分类或执行任务。

通过校验后调用现有 `BilibiliClient.delete_folders([fid])`。成功后再次拉取列表确认该 `fid` 已不存在；若仍存在则返回删除未生效错误。

## 错误处理

- 不存在：`FOLDER_NOT_FOUND`
- 默认收藏夹：`FOLDER_DELETE_PROTECTED`
- 非空收藏夹：`FOLDER_NOT_EMPTY`
- 有任务运行：`PIPELINE_RUNNING`
- B 站返回成功但收藏夹仍存在：`FOLDER_DELETE_NOT_CONFIRMED`

## 测试

- 后端覆盖成功删除、拒绝非空、拒绝默认收藏夹、删除后仍存在四种情况。
- 前端静态测试覆盖删除按钮条件、阻止冒泡、确认文案、DELETE 请求和刷新列表。
- 浏览器在桌面与 390px 手机宽度验证按钮不挤压标题和数量，并确认点击垃圾桶不会选中收藏夹。
