# 分类精细度与收藏夹资源列表设计

## 目标

1. 开始智能整理前允许选择“粗略 / 均衡 / 精细”，并支持自定义最终最大分类数，防止 AI 生成过多收藏夹。
2. 首页收藏夹右侧箭头进入独立资源列表，仅查看内容，不改变收藏夹多选状态。
3. 资源列表同时展示正常资源、失效资源，以及只能取得资源 ID/BVID 的不可访问资源。

## 分类精细度

### 首页控件

- 使用分段控件提供三个预设：粗略 `8` 类、均衡 `14` 类、精细 `24` 类。
- 默认选择均衡。
- 提供“自定义”入口，显示数值输入，范围 `3-50`；该数字表示最终方案允许的最大分类数。
- 开始按钮旁显示当前限制，例如“最多 14 个分类”。

### 会话与迁移

- `SessionIn` 新增 `category_limit: int`，范围 `3-50`，默认 `14`。
- `classify_sessions` 新增 `category_limit INTEGER NOT NULL DEFAULT 14`。
- 旧数据库通过 `_has_column()` 检查后执行 `ALTER TABLE`，已有会话自动使用 `14`，不重建表。
- `Storage.create_session()`、`ClassifySession.create()` 和 `create_many()` 传递并保存该值。

### AI 分类上限

- 每批分类提示词包含本次会话的最大分类数，鼓励批内复用稳定、宽泛的名称。
- 全部批次完成后统计全局分类名称；不超过上限时不增加 AI 调用。
- 超过上限时调用一次全局归并，要求把所有原分类映射到不超过 `category_limit` 个最终分类。
- 归并结果必须覆盖所有原分类，且最终唯一分类数不得超过上限；不满足时抛出 `AI_CATEGORY_LIMIT_FAILED`，不得悄悄创建超限方案。
- “未分类”不计入分类上限，也不参与目标收藏夹创建。

## 独立资源列表

### 路由与交互

- 新增视图 `folder-resources`，包含返回按钮、收藏夹名称、数量、加载状态、资源列表和“加载更多”。
- 收藏夹卡片继续负责多选；箭头改为独立按钮并阻止冒泡，仅进入资源列表。
- 列表条目显示封面、标题、UP 主、资源类型和状态，不提供删除、移动或编辑操作。
- 正常资源显示实际详情；失效资源显示“已失效”；完整 ID 清单中存在但详情分页从未返回的条目显示“无法访问”，并回退显示 BVID 或资源 ID。

### API

- 新增 `BilibiliClient.get_folder_resource_page(fid, page, page_size, storage)`，直接读取指定页，不从第一页重复扫描。
- 新增 `GET /api/folders/{fid}/resources?page=1&page_size=20`。
- 返回收藏夹信息、当前页原始资源、`has_more`、B 站声明总数；第一页额外返回 `/x/v3/fav/resource/ids` 的完整资源键。
- 前端累计各页已见资源键；最后一页加载完毕后，用完整资源键补齐“无法访问”条目。
- `page >= 1`，`page_size` 范围 `10-50`；收藏夹不存在时返回 `FOLDER_NOT_FOUND`。

## 错误与恢复

- 资源页请求失败时保留已经加载的内容，显示错误和“重试”按钮。
- 返回首页时重新拉取收藏夹，避免显示过期数量。
- 归并失败时沿用现有 SSE 失败事件，不保存超限分类方案。

## 验证

- 存储迁移、会话传参、批内提示词、全局归并上限及非法映射均有自动化测试。
- 资源 API 覆盖正常、失效、非视频和 ID-only 数据。
- 前端测试覆盖档位、自定义值、创建请求、箭头阻止冒泡、独立页面、分页和 ID-only 补齐。
- Playwright 检查桌面及 390px 手机宽度，不触发真实整理或资源修改。
